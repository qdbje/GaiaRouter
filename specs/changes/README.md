# 变更记录目录

本目录存放所有的规范变更记录。

## 目录结构

```
changes/
├── changelog.md          # 变更日志总览
├── proposals/            # 变更提案（待审批）
└── history/              # 历史变更记录
    ├── 2024/
    │   ├── 01/
    │   └── 02/
    └── ...
```

## 变更记录格式

### 变更日志条目格式

```markdown
## [变更ID] - YYYY-MM-DD

**变更类型**: [新增/修改/删除]
**影响范围**: [base/features/xxx]
**变更内容**: 
- 具体变更描述

**变更原因**: 
变更的原因和背景

**审批状态**: [待审批/已批准/已拒绝]
**审批人**: 
**实施日期**: 
```

## 变更流程

1. **提出变更**：在 `proposals/` 目录下创建变更提案
2. **评审变更**：团队评审变更提案
3. **批准变更**：变更批准后，更新到 `changelog.md`
4. **归档变更**：将已实施的变更归档到 `history/` 目录
5. **更新规范**：根据变更更新相应的规范文档

## 变更类型

- **新增**：新增规范或规则
- **修改**：修改现有规范或规则
- **删除**：删除过时的规范或规则
