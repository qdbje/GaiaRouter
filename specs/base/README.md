# 基础规则目录

本目录存放项目的基础规则和通用规范。

## 规则分类

### 编码规范
- 代码风格指南
- 命名规范
- 注释规范

### 架构规范
- 系统架构原则
- 设计模式规范
- 模块划分原则

### 接口规范
- API设计规范
- 数据格式规范
- 错误处理规范

### 测试规范
- 测试标准
- 覆盖率要求
- 测试用例编写规范

### 安全规范
- 安全要求
- 权限控制规范
- 数据保护规范

## 使用说明

1. 所有开发人员必须遵循本目录下的基础规则
2. 如需修改基础规则，请在 `../changes/` 目录下提交变更提案
3. 基础规则的变更需要经过团队评审和批准
