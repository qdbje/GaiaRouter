# 功能规范目录

本目录存放具体功能的规范文档，按功能模块组织。

## 目录结构

```
features/
├── feature-001/          # 功能001
│   ├── spec.md           # 功能规范文档
│   ├── requirements.md   # 详细需求
│   └── api.md            # API规范（如适用）
├── feature-002/          # 功能002
└── ...
```

## 功能规范模板

### spec.md 模板

```markdown
# 功能名称

## 功能概述

简要描述功能的目的和价值

## 需求描述

详细描述功能需求

## 输入输出

- 输入：描述输入参数和格式
- 输出：描述输出结果和格式

## 业务规则

列出相关的业务规则和约束

## 异常处理

描述异常情况和处理方式

## 验收标准

明确功能的验收标准

## 相关文档

- 设计文档：`../../designs/feature-xxx/`
- 任务分解：`../../tasks/feature-xxx/`
```

## 命名规范

- 功能目录命名：`feature-{编号}` 或 `{功能名称}`
- 规范文档命名：`spec.md`、`requirements.md`、`api.md` 等

## 工作流程

1. **创建功能规范**：在 `features/` 下创建功能目录和规范文档
2. **评审规范**：团队评审功能规范
3. **创建设计**：基于规范在 `../../designs/` 下创建设计文档
4. **分解任务**：在 `../../tasks/` 下分解开发任务
5. **实施开发**：在 `../../src/` 下实现功能
6. **编写测试**：在 `../../tests/` 下编写测试用例
