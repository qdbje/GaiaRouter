# 规范文档目录说明

本目录按照规范驱动开发（SDD）标准组织规范文档。

## 目录结构

```
specs/
├── base/           # 基础规则和通用规范
├── changes/        # 变更记录和变更日志
└── features/       # 功能规范（按功能模块组织）
```

## 目录说明

### 📁 base/ - 基础规则目录

存放项目的基础规则、通用规范、架构约束等，这些规则适用于整个项目：

- **编码规范**：代码风格、命名规范等
- **架构规范**：系统架构原则、设计模式等
- **接口规范**：API 设计规范、数据格式规范等
- **测试规范**：测试标准、覆盖率要求等
- **安全规范**：安全要求、权限控制等

### 📁 changes/ - 变更记录目录

存放所有的规范变更记录，按时间或版本组织：

- **变更日志**：记录每次规范变更的详细信息
- **变更提案**：待审批的规范变更提案
- **变更历史**：历史变更记录，便于追溯

### 📁 features/ - 功能规范目录

存放具体功能的规范文档，按功能模块组织：

```
features/
├── feature-001/     # 功能001的规范
│   ├── spec.md      # 功能规范文档
│   └── requirements.md  # 详细需求
├── feature-002/     # 功能002的规范
└── ...
```

## 工作流程

1. **定义基础规则**：在 `base/` 目录下创建项目的基础规则文档
2. **编写功能规范**：在 `features/` 目录下为每个功能创建规范文档
3. **记录变更**：当规范发生变更时，在 `changes/` 目录下记录变更详情
4. **更新规范**：根据变更记录更新相应的规范文档

## 规范文档模板

### 基础规则文档模板

- 规则名称
- 规则描述
- 适用范围
- 实施要求
- 示例说明

### 变更记录模板

- 变更 ID
- 变更日期
- 变更类型（新增/修改/删除）
- 变更内容
- 影响范围
- 变更原因
- 审批状态

### 功能规范模板

- 功能概述
- 需求描述
- 输入输出
- 业务规则
- 异常处理
- 验收标准
